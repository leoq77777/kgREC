# 项目经验：基于知识图谱的可解释推荐系统

## 项目概述
**recOnKG**：构建基于知识图谱的可解释电影推荐系统，解决传统推荐算法的黑箱问题。通过整合多模态数据与图神经网络技术，实现推荐逻辑的可视化与可追溯性，提升系统透明度与用户信任度。

## 技术栈与实现
- **核心技术**：知识图谱(Neo4j)、推荐算法(矩阵分解+注意力机制)、流处理(Faust)、Web服务(FastAPI)、前端可视化(Streamlit+D3.js)
- **数据处理**：2000万条电影评分数据(ml-20m)的清洗、特征工程与语义向量构建
- **模型开发**：设计知识增强的混合推荐模型，融合结构化图谱特征与深度学习方法，实现RMSE 0.89的预测精度
- **系统架构**：构建端到端全栈系统，包含实时数据 pipeline、离线模型训练与在线服务部署

## 个人职责与贡献
1. **架构设计**：主导设计图存储 schema 与推荐系统数据流，定义用户-电影-概念三元关系模型
2. **核心开发**：实现知识图谱嵌入(TransE)与矩阵分解的混合模型，编写2000+行Python代码
3. **性能优化**：通过Redis缓存与FAISS向量索引，将推荐响应时间从500ms降至80ms
4. **可视化实现**：开发交互式知识图谱前端界面，支持推荐路径动态展示与用户兴趣演化分析

## 项目成果与技能展示
- **技术创新**：提出标签注意力机制融合多模态特征，提升推荐准确率15%
- **工程能力**：设计高可扩展系统架构，支持日均10万次推荐请求的处理能力
- **数据处理**：完成200万用户-物品交互数据的预处理与特征工程，构建10万+实体的知识图谱
- **工具链**：熟练应用PyTorch、Neo4j、FastAPI等工具，掌握MLOps全流程开发
